Hash 뜻 잘게 자른 고기를 양파나 감자와 같이 다른 재료와 함께 튀겨 한 덩어리로 만든 요리 데이터를 입력받으면 완전히 다른 모습의 데이터로 바꾸어 놓는 작업

해시 테이블: 데이터의 해시 값을 테이블 내의 주소로 이용하는 궁극의 탐색 알고리즘 이진 탐색보다 빠름 암호화: 입력받은 데이터를 완전히 새로운 모습의 데이터로 만든다. 이 특성을 통해 암호화영역에서 중요하게 사용되고 있다. SHA(Secure Hash Algorithm)가 대표적 예 데이터 축약 : 길이가 서로 다른 입력 데이터에 대해 일정한 길이의 출력을 만들 수 있다. 이 특성을 이용하여 커다란 데이터를 해시하면 짧은 길이로 축약할 수 있다. 축약된 데이터끼리 비교하면 원본 데이터를 비교하는 것에 비해 엄청난 효율을 거둘 수 있다.

해시테이블: 공간을 팔아 시간을 사다.

배열을 선언하면 메모리(스택 or 자유 저장소)에 일정한 크기의 메모리가 예약된다. 그리고 배열 내 각 요소는 0, 1, 2와 같은 인덱스로 접근할 수 있다.

1번 인덱스는 Table[1]로 찾아가면 되는데 123817의 데이터는 어떻게 접근할까? 순차탐색이나 배열을 정렬한 뒤 이진탐색을 해야한다. 크기가 59999정도인 배열은 요즘의 컴퓨터는 사람이 감지하지못할정도의 아주 짧은 시간내에 탐색을 완료할 것이다. 평소에 사용하는 용도로면 아무 문제가 없을 것이다.

하지만 극한의 성능을 요구하는 분야에서는 이런 성능은 용납되지 않는다.

금융 분야같은 곳에선 시간이 금이다. 그렇기에 공간을 팔아 시간을 번다. 우선 순위의 문제이다. 123817(원본 데이터) → Hash function → 3819(테이블 내의 주소값) 이제 3819를 인덱스로 사용한다. Table[3819] = 123817;

데이터를 담을 테이블을 미리 크게 확보해 놓은 뒤, 입력받은 데이터를 해시하여 테이블 내의 주소를 확호하고 이 주소에 데이터를 담는다. ⇒ 해시 테이블 기본 개념

해시를 통해 데이터가 담길 주소값을 얻는 것은 상수 시간 내에 결정된다. 테이블 크기가 얼마나 커도 성능은 전혀 달라지지않는다. 배열과 다름없는 속도를 보여준다.
