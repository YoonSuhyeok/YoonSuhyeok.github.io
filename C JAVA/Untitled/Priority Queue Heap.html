Priority Queue & Heap

StatusPriority Queue &Heap & HashTable

핵심: 데이터의 입력/출력이 이루어질 때마다 최소한의 비용으로 최우선 순위의 데이터를 헤드에 위치시키는 알고리즘의 효율 ⇒ 나중에 배우는 그래프 알고리즘의 효율적 구현에 도움이 된다.

정의: 삽입(Enqueue), 제거( Dequeue) 연산을 지원하는 자료구조

우선 순위의 기준

우선 순위의 기준은 정해진 것이 없다. 이것을 어떻게 결정하는 것은 프로그래머에게 달려있다. 값이 클수록, 시간이 빠를수록, 이런 식으로 있다. 여기서는 값이 작을수록 우선순위로 높은 것으로 한다.

우선 순위 큐의 삽입 연산

각 요소가 우선순위를 가지고, 우선순위 오름차순을 가진다. 현재 이 책은 값이 작은 값이 우선순위가 높다고 정했다. 20 값을 삽입할 경우 우선 순위 첫요소부터 순차적으로 우선순위를 비교하고 2와 17사이에 삽입하게 된다. !문제는 순차탐색을 하면서 값을 삽입한다는 것

우선순위 큐의 제거연산

우선순위 큐의 제거(Dequeue) 간단하다. 맨 앞 값만 제거해서 반환하면 된다.

Heap

힙은 Heap Order Property을 만족하는 완전 이진트리를 말한다. 완전 이진 트리는 최고 깊이를 제외한 모든 깊이의 노드들이 완전히 채워져있는 트리를 말한다. Heap Order Property: 트리 내의 모든 노드가 부모 노드보다 커야한다는 규칙 루트 노드는 부모가 없어 예외이다. Q. 1. 이진 탐색이 가능한가요? 힙의 노드들은 자식 노드 > 부모 노드 때문에 안된다. Q 2. 특정 값을 찾으려면 어떻게 해야하나요? 찾기 위해서는 모든 노드를 순회해야한다.

힙에서 가장 작은 데이터를 갖는 노드는 Root 노드다!

힙은 두가지 연산만 가지고 있다. 노드 삽입, 루트 노드를 제거하는 최소값 삭제

힙에 새 노드 삽입하기

  1. 힙의 최고 깊이, 최 우측에 새 노드를 추가한다. 힙은 완전 이진 트리를 유지하도록 해야한다.
  1. 삽입한 노드를 부모 노드와 비교한다.

2-1. 삽입한 노드 > 부모 노드 : 값이 제대로 입력되었기에 연산을 종료한다. 2-2. 삽입한 노드 < 부모 노드 : 부모 노드와 삽입한 노드의 위치를 바꾸고 다시 2번을 반복한다.

힙의 최소값 삭제

힙의 최소값을 삭제한다. → root 노드 삭제 → Heap Order Property 유지

  1. 힙의 루트에 최고 깊이, 최 우측에 있던 노드를 루트 노드로 옮겨온다. 이때 힙의 힙 순서 속성이 파괴된다.
  1. 옮겨온 노드의 양쪽 자식을 비교하여 작은 쪽 자식과 위치 교환을 한다. 힙 순서 속성을 지키려면 부모 노드는 양쪽 자식보다 작은 값을 가져야한다. 그렇기에 작은 값과 교환한다.
  1. 옮겨온 노드가 더 이상 자식이 없는 잎 노드가 되거나 양쪽 자식보다 작은 값을 갖는 경우에 삭제 연산을 종료한다. 아니면 2번을 반복한다.

힙 구현

링크드 리스트로 구현할 경우 최고 깊이에 있는 최우측 노드를 찾기힘들다. 깊이 0 노드 → 배열의 0번 요소 깊이 1의 노드(모두 2개) → 배열 1~2번 요소에 저장 깊이 2의 노드(모두 4개) → 배열 3~6번 요소에 저장 깊이 N의 노드(2^n개) → 배열 2^n-1 ~ 2^(n+1)-2번 요소에 저장 K번 인덱스에 위치한 노드의 양쪽 자식 노드들이 위치한 인덱스 왼쪽 자식 노드 : 2k+1 오른쪽 자식 노드 : 2k+2 k번 인덱스에 위치한 노드의 부모 노드가 위치한 인덱스 : (k-1)/2의 몫

void HEAP_DeleteMin(Heap* H, HeapNode* Root){
	int ParentPosition = 0;
	int LeftPostion = 0;
	int RightPostion = 0;

	memcpy(Root, &H->Nodes[0], sizeof(HeapNode));
	memset(&H->Nodes[0], 0, sizeof(HeapNode));

	H->UsedSize--;
	HEAP_SwapNodes(H, 0, H->UsedSize);

	LeftPosition = HEAP_GetLeftChild(0);
	RightPostion = LeftPostion + 1;

	while(1){
		int SelectedChild = 0;
		
		if(LeftPostion >= H->UsedSize)
			break;
		if(RightPosition >= H->UsedSize){
			SelectedChild = LeftPosition;
		} else {
			if(H->Nodes[LeftPosition].Data > H->Nodes[RightPosition].Data)
				SelectedChild = RightPosition;
			else
				SelectedChild = LeftPosition;
		}
		if(H->Nodes[SelectedChild].Data < H->Nodes[ParentPosition].Data){
			HEAP_SwapNodes(H, ParentPostion, SelectedChild);
			ParentPostion = SelectedChild;
		} else 
			break;

		LeftPostion = HEAP_GetLeftChild(ParentPostion);
		RightPosition = LeftPosition + 1;
	}
	if(H->UsedSize < (H->Capacity/2){
		H->Capacity/=2;
		H->Nodes=(HeapNode*)realloc(H->Nodes,sizeof(HeapNode)*H->Capacity);
	}
}

힙을 이용한 우선순위 큐의 구현

힙은 최소값을 즉시 얻어낼수 있게한다는 점에서 우선순위 큐를 구현하는데 아주 적격인 자료구조다.

  • 노드 구조체에 Priority 필드 추가
  • 노드 구조체의 Data 필드 자료형을 void*로 변경
  • 삽입/삭제 함수의 이름을 HEAP_Insert()/HEAP_DeleteMin()에서 PQ_Enqueue()/PQ_Dequeue()로 변경

Data 필드 자료형을 void*로 변경한 것은 다양한 자료형을 우선순위 큐 안에 담을 수 있도록 하기 위해서이다. 그 예(최소신장트리, 다익스트라의 최단 경로 알고리즘, 허프만 압축 알고리즘 등)
