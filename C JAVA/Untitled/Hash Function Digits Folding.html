자릿수 접기

극한의 성능을~!!! 나눗셈은 서로 다른 입력 값에 대해 동일한 해시 값, 즉 해시 테이블 내의 동일한 주소를 반환할 가능성이 높다. 이것을 충돌(Collision)이라 한다. 똑같은 해시 값이 아니더라도 해시 테이블 내의 일부 지역의 주소들을 집중적으로 반환하는 결과로 데이터들이 한 곳에 모이는 문제가(이를 클러스터(Cluster)라한다.)발생한 가능성이 높다.

충돌이나 클러스팅의 문제를 완벽하게 제거할 수 있는 해싱 알고리즘은 존재하지 않는다. 하지만 문제를 일으킬 가능성을 줄인 알고리즘이 있다. 자릿수 접기이다.

ex) 문방구에서 전지를 한장을 샀다고 생각해보자. 전지를 반으로 접으면 2절지가, 2절지를 반으로 접으면 4절지가 된다. 또 접으면 8절지가 된다.

같은 이치로 자릿수 접기는 종이를 접듯이 숫자를 접어 일정한 크기 이하의 수로 만드는 방법이다. 예) 8 1 2 9 3 3 5 → 각 자릿수를 더하면 31이다 31 = 8 + 1 + 2 + 9 +3 + 3 + 5 ⇒ 8129335가 31로 접혔다. 이제는 두자리씩 더해보자 148 = 81 + 29 + 33 + 5 8129335가 148로 접혔다.

이 역시 일정한 범위 내의 해시 값을 얻을 수 있다. 10진수의 경우 모든 수는 각 자리마다 0~9까지의 값을 가질 수 있으니, 7자리수에 대해 한자리씩 접기를 하는 경우 최소 0에서 최대 63(9+9+9+9+9+9+9)까지의 해시 값을 얻을 수 있고, 두 자리씩 접을 경우 최소 0에서 최대 306(99+99+99+9)까지의 해시 값을 얻을 수 있다.

자릿수 접기는 문자열을 키로 사용하는 해시 테이블에 특히 잘 어울리는 알고리즘이다. 문자열의 각 요소를 ASCII코드 번호(이 번호는 0~127사이의 값을 가진다.)로 바꾸고, 이 값을 각각 더하여 접으면 문자열을 깔금하게 해시 테이블의 주소로 바꿀 수 있다.

아래는 문자열 키를 자릿수 접기 알고리즘을 통해 해시값을 만들어내는 코드다.

int Hash(Char* Key, int KeyLength, int TableSize){
	int i = 0;
	int HashValue = 0;

	// 문자열의 각 요소를 ASCII 코드 번호로 변환하여 더한다.
	for( i=0; i<KeyLength; i++)
		HashValue += Key[i];

	return HashValue % TableSize;
}

간단하게 함수를 짤 수 있다. 하지만 문제가 있다 해시 테이블의 크기가 12289이고 문자열 키의 최대 길이가 10자리라고 한다면, 해시 함수는 10*127 = 1270(최대 값)이므로 0에서 1270사이의 주소만을 반환하게 되여 1271~12288 사이의 주소가 전혀 활용되지 않게 된다. ASCII 코드로 10자리를 만들었을 때 조합할 수 있는 경우의 수가 127^10 = 1,091,533,853,073,393,531,649가지나 된다는 사실을 생각하면 문제라고 생각될 수 밖에 없다.
